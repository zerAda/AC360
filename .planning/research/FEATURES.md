# FEATURES.md — Client Assistant Features

## Table Stakes

Must-have features or users will not adopt:

| Feature | Description | Complexity |
|---------|-------------|-----------|
| Natural language queries | Understand freeform questions | Medium |
| Folder summarization | Summarize client folder in 2-3 sentences | Low |
| Document search | Find specific documents by description | Low |
| Permission enforcement | Never show content user can't access | High |
| "Not found" response | Clearly say when information unavailable | Low |
| Source citation | Cite documents used in response | Low |

## Differentiators

Competitive advantage features:

| Feature | Description | Complexity |
|---------|-------------|-----------|
| Meeting preparation | Auto-generate meeting summary | Medium |
| Risk identification | Identify attention points | Medium |
| Email drafting | Generate follow-up email drafts | Medium |
| Clause extraction | Extract specific contract clauses | High |

## Anti-Features

Features to NOT build in POC:

| Feature | Reason |
|---------|--------|
| Multi-SharePoint search | Out of scope |
| CRM integration | Not in Scenario A |
| Real-time notifications | Not required |
| Analytics dashboard | Post-POC |
| Custom workflows | Not in POC scope |
