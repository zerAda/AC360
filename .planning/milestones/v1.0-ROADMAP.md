## Phases

### Phase 1: UI/UX et Contrôle Audit

**Goal:** Ajouter le bouton d'annulation et la barre de progression

**Requirements:** UI-01, UI-02, UI-03, UI-04, UI-05

**Success Criteria:**
1. Bouton "Annuler" visible et fonctionnel pendant l'audit
2. L'audit s'arrête proprement quand on clique annuler (flag d'interruption)
3. Barre de progression visible avec % d'avancement
4. Progression mise à jour en temps réel (pas de blocage UI)
5. Message de confirmation à la fin

**Files:** `src/main.py` (AuditApp class)

---

### Phase 2: Nettoyage de Données

**Goal:** Dissocier IBAN/Nom, filtrer bordereaux et récaps

**Requirements:** DATA-01, DATA-02, DATA-03, DATA-04, DATA-05, FILT-01, FILT-02, FILT-03, FILT-04, FILT-05

**Success Criteria:**
1. IBAN et nom de société séparés depuis le PDF (IBAN = positions 0-27, nom = 28+)
2. Noms normalisés (accents, majuscules, préfixes/suffixes)
3. Correspondance fuzzy fonctionne avec seuil 75%
4. Lignes "bordereau" exclues de l'analyse Excel
5. Lignes de récap/sommes exclues ("Total", "Somme", "Récap")
6. Lignes sans IBAN valide exclues du PDF

**Files:** `src/core.py` (PDFParser, ExcelParser, Filters, Normalizer, FuzzyMatcher)

---

### Phase 3: Optimisation Performance

**Goal:** Accélérer le traitement avec des algorithmes optimisés (< 30s pour 50+ virements)

**Requirements:** PERF-01, PERF-02, PERF-03

**Success Criteria:**
1. Traitement de 50+ virements en moins de 30 secondes (benchmark intégré)
2. Interface reste réactive pendant l'audit (threading)
3. Pas de surcharge mémoire (gc.collect())

**Files:** `src/core.py` (OptimizedMatcher), `src/main.py` (threading, benchmark)

---

### Phase 4: Calcul des Écarts

**Goal:** Calculer les écarts entre Excel et PDF, corriger le total

**Requirements:** CALC-01, CALC-02, CALC-03, CALC-04, CALC-05, CALC-06

**Success Criteria:**
1. Écart calculé pour chaque société : Écart = Excel − PDF
2. Écart positif = Excès Excel, Écart négatif = Manque Excel
3. Total d'écart = somme des écarts individuels
4. Total écran Dashboard cohérent avec total calculé (vérification automatique)
5. Source de vérité = PDF respectée

**Files:** `src/core.py` (EcartCalculator)

---

### Phase 5: Post-Audit

**Goal:** Vider les fichiers et permettre un nouvel audit propre

**Requirements:** POST-01, POST-02, POST-03, POST-04

**Success Criteria:**
1. PDF uploadés supprimés après audit terminé (sécurisé)
2. Excel uploadés supprimés après audit terminé (sécurisé)
3. Nouvel audit peut être lancé proprement (réinitialisation complète)
4. Barre de progression et résultats réinitialisés

**Files:** `src/main.py` (_safe_delete_file, reset_all)

## Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Source de vérité = PDF | Le PDF est le bordereau bancaire officiel | ✅ Validated v1.0 |
| Seuil fuzzy = 75% | Valide par le métier pour tolérer les variantes de noms | ✅ Validated v1.0 |
| Écart = Excel − PDF | Montre si Excel a plus ou moins que la réalité bancaire | ✅ Validated v1.0 |
| Interface en français | Utilisateurs francophones | ✅ Validated v1.0 |
| Exclure lignes "bordereau" | Ce sont des tests/calculs des gestionnaires | ✅ Validated v1.0 |
| Algorithme O(n) avec index | Performance critique pour 50+ virements | ✅ Validated v1.0 |
