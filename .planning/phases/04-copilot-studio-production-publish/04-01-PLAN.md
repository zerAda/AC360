<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo `.mcs.yml` → live Copilot Studio agent | Declarative guardrail/endpoint config in git is the source of truth the CI validator enforces before it is published to the live agent (untrusted-config-drift boundary). |
| topic HttpRequestAction → backend gateway | The URL string in the topic determines which gateway (staging vs prod) receives the user's Bearer token — a cutover-integrity boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | settings.mcs.yml guardrail keys | mitigate | `find_agent_guardrail_issues` fails CI if `useModelKnowledge` is not exactly false or agent `contentModeration` is not High (PUB-04); pytest pins both. |
| T-04-02 | Information Disclosure | model "general knowledge" presented as client fact | mitigate | `useModelKnowledge=false` asserted offline (grounded-only); validator + Wave-0 test enforce it stays false. |
| T-04-03 | Tampering | stale staging gateway URL shipped to prod | mitigate | Staging host added to `find_wiring_issues` fail-closed check → any remaining `ac360-gateway-staging` fails CI (PUB-02); acceptance grep proves zero remain. |
| T-04-04 | Spoofing | Bearer token forwarded to wrong gateway audience | mitigate | URLs rebound to the deterministic prod host only; the prod API audience scope is NOT hardcoded in repo (bound in agent UI per Plan 02) — no wrong-audience GUID can leak via these files. |
| T-04-05 | Tampering | structural edit of topic nodes breaks solution import | accept | Edit restricted to the host substring only (RESEARCH anti-pattern); no node restructuring — low residual risk for a value-only change. |
| T-04-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan (validator uses already-pinned pyyaml; tests use already-pinned pytest) — supply-chain surface is nil this plan. |
</threat_model>
