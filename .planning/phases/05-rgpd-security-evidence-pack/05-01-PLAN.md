## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Durable Functions VM-local FS → operator/runtime | `JOBS_BASE_DIR` holds downloaded client documents, OCR temp, FIC drafts (PII) on ephemeral storage |
| Azure Storage account → tenant | Same account holds Durable control/lease blobs AND (potentially) job/OCR/FIC artifact blobs |
| Bicep template → live Azure resource | Retention params control how long PII is recoverable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | PII over-retention on JOBS_BASE_DIR | mitigate | Age-based `prune_jobs_dir` daily timer deletes artifacts > 30d; pure/tested; documented manual fallback in RGP-03 policy doc |
| T-05-02 | Information Disclosure | PII over-retention in Storage blobs | mitigate | `managementPolicies` server-side delete @ 30d for job/OCR/FIC blobs |
| T-05-03 | Denial of Service | Lifecycle rule deletes Durable task-hub control/lease blobs → orchestration corruption | mitigate | `filters.prefixMatch: jobBlobPrefixes` scopes the rule to job blobs only — never account-wide (RESEARCH Pitfall 3) |
| T-05-04 | Tampering | `prune_jobs_dir` deletes in-flight (fresh) audit artifacts | mitigate | Age-based cutoff (mtime < now - max_age); fresh entries kept; NOT the destructive full-wipe script (RESEARCH Anti-Pattern) |
| T-05-05 | Repudiation | Erasure window misstated (soft-delete/versioning hides ~7 extra days) | mitigate | RGP-03 doc states honest ~37-day effective-erasure window; snapshot/version delete actions in the rule |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase (RESEARCH Package Legitimacy Audit: not applicable — slopcheck no-op) |
