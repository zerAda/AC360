<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → Entra ID | OIDC token exchange (federated trust, no stored secret) |
| GitHub Actions → Azure control plane | deploy identity mutates rg-ac360-prod resources |
| git tag push → production deploy | a tag can trigger the pipeline up to the approval gate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-13 | Spoofing/Elevation | stolen long-lived deploy secret | mitigate | OIDC federated credential — no stored SP secret; subject scoped to repo+environment (azure/login@v2, id-token: write) |
| T-03-14 | Elevation | over-privileged deploy identity | mitigate | Deploy identity granted least-privilege role on rg-ac360-prod only (operator step, OPS-01) — not subscription Owner |
| T-03-15 | Tampering | unreviewed prod deploy | mitigate | GitHub production Environment required reviewer + what-if diff gate before apply |
| T-03-16 | Spoofing | federated subject mismatch (what-if vs deploy job) | mitigate | Distinct federated subjects: deploy job :environment:production; what-if job tag/PR subject — resolved as the OPS-01 decision (Pitfall 5); documented in YAML comment + runbook |
| T-03-SC | Tampering | supply-chain (actions) | mitigate | First-party Azure/* and actions/* pinned to tags; existing CI gitleaks/pip-audit jobs unchanged |
</threat_model>
