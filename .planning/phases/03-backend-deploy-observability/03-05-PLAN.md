| operator → prod app settings | feature-flag kill-switch mutates live consumption gates |

| runbook docs → repo | runbooks must not embed live secrets or cleartext UPNs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-17 | Information Disclosure | secrets/PII embedded in runbook examples | mitigate | Runbooks use placeholders (<ORG>/<REPO>, <hash>, OBO-CLIENT-SECRET name only); user blocking uses hash_id, never cleartext UPN (analog EMERGENCY_SHUTDOWN_RUNBOOK lines 51-54) |
| T-03-18 | Elevation | kill-switch / rotation by non-operator | mitigate | App settings + Key Vault mutable only by an admin with resource access (analog Principe); actions traced (admin_controls) |
| T-03-19 | Tampering | rollback to a tampered/old tag | mitigate | Rollback target = an immutable signed prod-* git tag (known-good marker); what-if diff visible before any re-apply |
| T-03-20 | Repudiation | untracked secret expiry → silent outage | mitigate | OPS-03 expiry-tracking table (per secret next-rotation date); OIDC removes the long-lived deploy secret entirely |
</threat_model>
