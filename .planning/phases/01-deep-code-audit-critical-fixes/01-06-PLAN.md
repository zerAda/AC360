<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller -> audit job ownership | A caller must only read jobs whose persisted owner_hash matches hash_id(their oid); the durable check is authoritative (AUD-03). |
| gateway -> OBO endpoint | Transient OBO failure must surface as 503 (retriable), not 502, and never leak token/exception detail (AUD-05/06). |
| app -> response body & telemetry | HTTPException detail and telemetry dimensions cross to the client and to App Insights; both must be redacted (AUD-06). |
| activity boundary | No file path produced in one activity may be consumed in another (AUD-08). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06-01 | Elevation of privilege / Info disclosure | audit-job IDOR | mitigate | hash_id(oid) at all 3 sites; _assert_durable_owner hard-fails on mismatch as the authoritative gate; fast-path map is cache only; optional fail-closed on absent owner_hash for completed jobs (clean cutover). |
| T-01-06-02 | Denial of Service / availability | OBO exhaustion status | mitigate | Route through retrying wrapper; 503 (retriable) on exhaustion, not a misleading 502; generic redacted detail. |
| T-01-06-03 | Information disclosure | response detail + telemetry dims | mitigate | Every dynamic detail + AppInsightsMiddleware dimension routed through the safe_logger redaction helper; generic message + correlation id to client (Pitfall 5). |

| T-01-06-05 | Repudiation | document-access trail | mitigate | emit_document_access fired on the access path (oid-bearing site) with the 4-field PII-free record (AUD-07). |
