## Directory Purposes

**`src/copilot/AC360/`:**
- Purpose: Copilot Studio bot definition exported to repo (PAC format)
- Contains: Agent config, 25+ topics, knowledge sources, connection refs, action definitions
- Key files: `agent.mcs.yml` (main), `topics/*.mcs.yml` (flows), `settings.mcs.yml` (config)
- Committed: Yes (source of truth for bot behavior)
- Generated: No (manually edited, exported from Copilot Studio)

**`azure_functions/`:**
- Purpose: Backend orchestration service (Durable Functions v2, Python)
- Contains: Orchestration logic, activity definitions, shared utility modules
- Key files: `function_app.py` (entry), `shared/audit_pipeline.py` (pure logic), `shared/sharepoint.py` (Graph I/O)
- Committed: Yes
- Generated: No

**`scripts/`:**
- Purpose: Application server, business logic, and utilities (FastAPI + pure functions)
- Contains: FastAPI server (api_server.py), auth, audit engine, Fabric query builder, FIC generation, admin tools
- Key files: `api_server.py` (646 lines, main server), `auth.py` (JWT validation), `fabric_audit_engine.py` (comparison logic)
- Committed: Yes
- Generated: No

**`tests/`:**
- Purpose: Comprehensive test suite (30+ files, security + logic + integration focused)
- Contains: Unit tests, mocks, integration tests, acceptance matrices, red team prompts
- Key files: `backend/test_auth_jwt.py`, `backend/test_audit_ownership.py`, `azure_functions/test_audit_pipeline.py`
- Committed: Yes
- Generated: No

**`schemas/`:**
- Purpose: JSON Schema validation contracts for audit inputs/outputs and admin actions
- Contains: `.schema.json` files for OCR results, audit comparison inputs/outputs, admin controls
- Committed: Yes
- Generated: No (manually maintained as contracts evolve)

**`docs/`:**
- Purpose: Enterprise documentation (architecture, security, governance, runbooks, guides)
- Contains: 50+ markdown files covering all aspects (architecture, security, deployment, incident response)
- Committed: Yes
- Generated: No (maintained by team)

**`infra/`:**
- Purpose: Infrastructure as Code (Bicep) for hardened Azure resources
- Contains: `main.bicep` (resource definitions), `staging.parameters.json` (parameters)
- Committed: Yes
- Generated: No

**`.planning/codebase/`:**
- Purpose: Generated by `/gsd-map-codebase` (this tool)
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md (quality focus only), TESTING.md (quality focus only)
- Committed: Yes (generated on demand)
- Generated: Yes

## Key File Locations

**Entry Points:**
- `scripts/api_server.py` – FastAPI server (HTTP interface); launch with `uvicorn scripts.api_server:app`
- `azure_functions/function_app.py` – Durable Functions orchestration (Azure runtime only)
- `src/copilot/AC360/agent.mcs.yml` – Copilot Studio bot definition (published via PAC CLI)

**Configuration:**
- `scripts/config.py` – Centralized AppConfig (TENANT_ID, CLIENT_ID, JOBS_BASE_DIR, allowed extensions, scopes, roles)
- `.env` (local dev) or Azure App Settings (production) – Environment variables (never committed)
- `azure_functions/local.settings.json.example` – Template for local Function dev secrets
- `infra/staging.parameters.json` – IaC parameters (fill before `az deployment group create`)

**Core Logic:**
- `azure_functions/shared/audit_pipeline.py` – Pure orchestration (testable, no cloud calls)
- `scripts/fabric_audit_engine.py` – Field aliasing, normalization, verdict logic
- `scripts/auth.py` – JWT validation, JWKS caching, scope/role verification
- `azure_functions/shared/sharepoint.py` – Secure document download with guards

**Testing:**
- `conftest.py` (root) – Pytest configuration, test environment injection
- `tests/backend/conftest.py` – Backend test setup (adds scripts/ to path, stubs env vars)
- `tests/backend/test_auth_jwt.py` – JWT validation tests
- `tests/backend/test_audit_ownership.py` – IDOR prevention tests
- `tests/azure_functions/test_audit_pipeline.py` – Audit orchestration tests (mocked)

## Naming Conventions

**Files:**
- **Python modules (scripts):** `snake_case.py` (e.g., `api_server.py`, `fabric_audit_engine.py`)
- **Test files:** `test_*.py` (e.g., `test_auth_jwt.py`, `test_path_traversal.py`)
- **Copilot topics:** `PascalCase.mcs.yml` (e.g., `ConversationStart.mcs.yml`, `PreparationRDVRenouvellement.mcs.yml`)
- **Config files:** Lowercase with underscores (e.g., `.env.example`, `local.settings.json.example`)
- **Schema files:** `snake_case.schema.json` (e.g., `audit_result.schema.json`)
- **PowerShell scripts:** `PascalCase.ps1` (e.g., `sync_copilot.ps1`, `package_release.ps1`)

**Directories:**
- **Python packages:** `lowercase` (e.g., `scripts/`, `tests/`, `schemas/`)
- **Copilot groups:** `PascalCase` (e.g., `topics/`, `knowledge/`, `actions/`)
- **Logical groupings:** `snake_case` (e.g., `.planning/`, `.github/`)
- **Component names:** Match repo structure (e.g., `azure_functions/`, `infra/`, `docs/`)

**Functions & Classes:**
- **Functions:** `snake_case()` (e.g., `run_audit()`, `download_document()`, `verify_azure_ad_token()`)
- **Classes:** `PascalCase` (e.g., `AppConfig`, `AuditDeps`, `FastAPI`)
- **Constants:** `SCREAMING_SNAKE_CASE` (e.g., `JWKS_TTL_SECONDS`, `_RATE_LIMIT_MAX`)
- **Private functions:** Prefix with `_` (e.g., `_safe_filename()`, `_fetch_jwks()`)

**Variables:**
- **User-facing:** `upn` (user principal name), `document_id` (UUID), `job_id` (orchestration ID)
- **Hashed identifiers:** `*_hash` (e.g., `owner_hash`, `user_id_hash` for privacy)
- **Configuration:** Uppercase with underscores (e.g., `TENANT_ID`, `JOBS_BASE_DIR`, `AZURE_FUNCTION_URL`)

## Where to Add New Code

**New API Endpoint:**
- **Implementation:** `scripts/api_server.py` – Add route handler (e.g., `@app.post("/api/new-action")`)
- **Auth:** Use dependency injection `user_upn: str = Depends(verify_azure_ad_token)`
- **Rate limit:** Call `await _check_rate_limit(user_upn)` for operations needing quota
- **Feature gate:** Call `is_allowed("new-feature", user_id_hash=hash_id(user_upn))`
- **Tests:** Create `tests/backend/test_new_endpoint.py` with mocked dependencies

**New Business Logic:**
- **Location:** `scripts/` as a new module (e.g., `scripts/new_processor.py`)
- **Pattern:** Pure functions with dependency injection; no direct cloud SDK imports
- **Testing:** `tests/backend/test_new_processor.py` with fake dependencies (thefuzz, pydantic for validation)
- **Integration:** Imported and called from `api_server.py` or `azure_functions/shared/audit_pipeline.py`

**New Audit Stage (Durable Activity):**
- **Orchestration:** Define in `azure_functions/function_app.py` (activity function + orchestrator step)
- **I/O:** Implement logic in `azure_functions/shared/` module (e.g., `new_activity.py`) as pure function
- **Injection:** Add to `AuditDeps` if testable dependency needed (e.g., `new_processor()` callable)
- **Tests:** Mock the dependency in `tests/azure_functions/test_audit_pipeline.py`

**New Topic (Copilot):**
- **Location:** `src/copilot/AC360/topics/NewTopic.mcs.yml`
- **Pattern:** Follows Copilot Studio YAML format; references existing actions or endpoints
- **Deployment:** Sync via `scripts/sync_copilot.ps1 -Push`
- **Testing:** Manual testing in Copilot Studio or via `scripts/run_demo.py`

**New Test Case:**
- **Security focus:** `tests/backend/test_security_*.py` (e.g., `test_injection_*.py`)
- **Logic focus:** `tests/backend/test_feature_*.py` or `tests/azure_functions/test_*.py`
- **Fixtures:** Define in `conftest.py` (e.g., `@pytest.fixture def valid_jwt():`)
- **Mocking:** Use `unittest.mock.patch` for cloud SDK calls

**New Schema:**
- **Location:** `schemas/new_contract.schema.json`
- **Format:** JSON Schema draft-7 or 2020-12
- **Validation:** Automatic in `azure_functions/shared/audit_pipeline.py` via `_validate_schema()`
- **Docs:** Update `docs/architecture/ARCHITECTURE.md` to reference new contract

**New Utility Script:**
- **Admin scripts:** `scripts/new_admin_task.ps1` (PowerShell for SharePoint/Azure setup)
- **Python utilities:** `scripts/new_utility.py` (e.g., data export, reporting)
- **CI gates:** `scripts/validate_*.py` (e.g., validation checks in pipeline)
- **Placement:** Root of `scripts/` or `scripts/` subdirectory if related to a feature
